# 辩论工作流编排指南

本 skill 指导主 agent 如何编排辩论工作流。各子智能体的具体行为和输出格式已在各自的 agent 定义中详细描述，此处仅关注编排逻辑。完整的架构图和目录结构见 `AGENTS.md`。

## 第一步：初始化

生成工作目录 `debates/{YYYYMMDD_HHmmss}_{topic_slug}/`，其中 `topic_slug` 从用户问题中提取 2-4 个关键词，用下划线连接。

```bash
mkdir -p debates/{id}/rounds
```

## 第二步：问题分析

调用 `debate-analyzer`：

```
请分析以下辩论问题，并将结果写入 {work_dir}/analysis.md：

问题：{用户的问题}

辩论工作目录：{work_dir}
```

如果用户提交了附件（图片、文档等），在 prompt 中一并说明。

## 第三步：辩论循环

默认正方先发言。最多 20 轮。

每轮执行三步：**先发言方 -> 后发言方 -> 评判**。

### 调用先发言方

```
请进行第 {round_num} 轮辩论的{side}方发言。
辩论工作目录：{work_dir}
轮次：{round_num}
你是本轮先发言方（上一轮优势方）。
{第一轮: "这是第一轮，请阐述你的初始立场。"}
{后续轮次: "对方上一轮发言：" + 读取上一轮对方发言文件内容}
```

### 调用后发言方

```
请进行第 {round_num} 轮辩论的{side}方发言。
辩论工作目录：{work_dir}
轮次：{round_num}
对方本轮发言：{读取本轮先发言方的发言文件内容}
```

### 调用评判

```
请对第 {round_num} 轮辩论进行评判。
辩论工作目录：{work_dir}
轮次：{round_num}
```

### 解析评判结果

读取 `rounds/round_{N}_judge.md`，从「判定」部分提取：
- **是否继续辩论** -> 如果为「否」则跳出循环
- **下一轮先发言方** -> 更新下一轮的发言顺序

也可以读取 `state.md` 中的 `下一轮先发言方` 字段，效果相同。

## 第四步：总结陈词

辩论结束后，**依次**调用正方和反方做总结陈词：

```
辩论环节已结束，请做总结陈词。
辩论工作目录：{work_dir}
请将总结陈词写入 {work_dir}/closing_{side}.md
```

## 第五步：生成报告

调用 `debate-summarizer`：

```
辩论已结束，请生成最终总结报告。
辩论工作目录：{work_dir}
最后一轮轮次：{last_round_num}
```

## 第六步：返回结果

读取 `{work_dir}/final_report.md`，将内容呈现给用户。

## 编排要点

- 使用 `TodoWrite` 跟踪每一步进度，每完成一步及时更新状态
- 每轮辩论中三个子智能体必须**串行**调用（后一个依赖前一个的输出）
- 解析 judge 输出时，关注 `是否继续辩论` 和 `下一轮先发言方` 两个字段即可
- 如果中途中断，读取 `state.md` 恢复进度，从上次的轮次和阶段继续
- 向用户报告进度时，简要说明当前轮次和阶段，不需要复述辩论内容
